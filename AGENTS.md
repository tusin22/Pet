# AGENTS.md — Instruções operacionais obrigatórias

Escopo: todo o repositório.

## Regra principal (obrigatória)
Antes de alterar qualquer código ou conteúdo, o agente **deve ler este arquivo** e seguir estas regras.

## Regras de atualização
1. Sempre registrar o que foi feito no `CHANGELOG.md` na seção **[Não lançado]**.
2. Toda alteração deve ser descrita de forma objetiva (arquivo alterado + resumo).
3. Se criar novo arquivo relevante, documentar no `README.md` quando aplicável.
4. Evitar mudanças grandes sem plano: dividir em etapas pequenas e verificáveis.
5. Em qualquer entrega, priorizar estabilidade do MVP aprovado.

## Padrão de registro no CHANGELOG
- Tipo: `Adicionado`, `Alterado`, `Corrigido`, `Removido`.
- Texto curto e direto, com foco em impacto funcional.

## Objetivo do projeto
Colocar o site do petshop em produção com qualidade, mantendo clareza de documentação e histórico das mudanças.
